\chapter{Introdução}
\label{capitulo:introducao}

O xadrez é um dos jogos de tabuleiro mais antigos e estudados do mundo, sendo frequentemente utilizado ao longo da história como um laboratório para testar a capacidade lógica e o poder de processamento dos computadores. Desde a publicação do artigo seminal de Shannon (1950), o xadrez computacional tem servido como um campo fértil para explorar os limites da inteligência artificial. Nesse sentido, segundo o \citeonline{USP2025}, o computador assume o papel de uma "máquina epistemológica", auxiliando na construção coletiva do conhecimento sobre o jogo e servindo de laboratório para algoritmos de alta complexidade.

Nos primórdios da ciência da computação, as partidas contra a máquina eram restritas a terminais de texto ou ambientes digitais extremamente rudimentares. No entanto, com a evolução tecnológica e a mudança nas expectativas dos usuários, os softwares de entretenimento passaram a exigir não apenas uma lógica matemática impecável, mas também uma camada de visualização imersiva e responsiva.

Atualmente, o desenvolvimento desse tipo de sistema envolve um desafio de engenharia de software bastante específico: integrar um arranjo visual atraente, rico em animações, a um núcleo de processamento capaz de calcular milhões de probabilidades matemáticas em poucos segundos. 

Motores de jogos modernos, como a Godot Engine, são ferramentas excelentes para gerenciar sistemas interativos e efeitos sonoros. Contudo, não foram projetados para processar nativamente algoritmos densos de inteligência artificial. Por outro lado, ferramentas consagradas como o Stockfish são extremamente poderosas, mas operam exclusivamente via linha de comando, sem oferecer componentes de interface para o jogador humano.

Diante desse cenário, este trabalho apresenta o desenvolvimento do simulador digital \textit{Royal Legacy}. O problema de pesquisa que norteia este projeto é: como integrar de forma eficiente um motor lógico de xadrez de alta complexidade (Stockfish) a uma estrutura gráfica moderna (Godot Engine), garantindo que o sistema seja visualmente personalizável e que os cálculos da máquina não comprometam a fluidez da experiência do usuário durante as partidas

\section{Objetivo Geral}

Este trabalho tem como objetivo desenvolver um simulador de xadrez digital interativo e customizável na Godot Engine, integrado ao motor de inteligência artificial Stockfish através de uma comunicação em linguagem Python, proporcionando uma experiência de jogo fluida, modular e com diferentes níveis de dificuldade.

\section{Objetivos específicos}

Para alcançar o objetivo geral, foram definidos os seguintes objetivos específicos: 

\begin{itemize}
	\item Implementar a camada gráfica e a lógica completa das regras do xadrez utilizando a linguagem nativa GDScript;
	\item Desenvolver um \textit{middleware} em Python para estabelecer a comunicação assíncrona entre o \textit{frontend} e o binário do Stockfish, operando através do protocolo \textit{Universal Chess Interface} (UCI);
	\item Estruturar um sistema de gerenciamento de cenas e menus de opções, garantindo uma experiência de usuário (UX) intuitiva;
	\item Criar um sistema dinâmico de temas visuais baseado em dicionários de dados, permitindo a alteração estética em tempo real;
	\item Desenvolver um gerenciador de estados para controlar turnos e condições de fim de jogo, validando o fluxo correto das partidas nas modalidades Jogador contra Jogador (PvP) e Jogador contra Máquina (PvE).
\end{itemize}

\section{Justificativa}
\label{secao:justificativa}

A relevância deste trabalho reside na demonstração prática de uma arquitetura de software altamente modular. Conforme defende \citeonline{Fowler2006}, a separação entre a apresentação e o modelo é uma das heurísticas fundamentais do projeto de software, permitindo que a lógica de domínio evolua independentemente da interface. Este projeto propõe uma ponte técnica entre tecnologias de naturezas distintas, resolvendo uma lacuna comum: a dificuldade de executar processos pesados em segundo plano sem impactar a fluidez visual. 

Ao documentar a separação dessas responsabilidades, este trabalho serve como um modelo de estudo valioso para a integração de motores visuais a inteligências artificiais complexas, evitando travamentos de tela comuns em implementações que utilizam apenas uma única via de processamento (\textit{thread}).

\section{Contribuições realizadas}

O desenvolvimento deste simulador trouxe contribuições práticas e metodológicas, destacando-se:

\begin{itemize}
	\item \textbf{Arquitetura de Comunicação Assíncrona:} A implementação de um \textit{script} em Python que atua como ponte provou ser viável para acoplar programas externos pesados à Godot sem causar interrupções (\textit{stutters}) na interface principal.
	\item \textbf{Modularidade de Sistema:} A centralização da navegação por meio de um gerenciador global demonstrou a aplicação de boas práticas de \textit{design}, facilitando a manutenção e evitando o acoplamento excessivo.
	\item \textbf{Escalabilidade Visual:} O uso de estruturas de dados flexíveis resultou em um código mais limpo e otimizado para futuras expansões de conteúdo.
\end{itemize}
